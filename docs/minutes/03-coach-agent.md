# Debate Coach Agent · 策略决策层级分析

> **发言时间**: SESSION-001, 第三席
> **视角**: 决策权在谁手里的权力模型
> **核心问题**: 谁来决定"这场要怎么打"？决策权在谁手里？

---

## 1. 教练不是一个人，是一个决策链

"教练"听起来是单一角色，但在架构上是**至少 4 个独立的决策节点**：

```
决策节点 1: Pre-debate Coach (赛前)
  输入: 辩题 + 己方模型池 + 对方历史数据
  输出: strategy_doc (全局策略)
  决策权: ★★★★ (4/5 可以决定比赛走向)

决策节点 2: Round Whisper Coach (轮间)
  输入: 当前比赛进展 + strategy_doc
  输出: round_whisper (轮次微调)
  决策权: ★★ (2/5 只能在 strategy_doc 框架内调整)

决策节点 3: Debater Router (辩手路由)
  输入: 当前轮次 + 需要辩论的角色
  输出: 调用哪个模型、什么参数
  决策权: ★★ (2/5 技术选择, 不涉及策略)

决策节点 4: Emergency Coach (紧急情况)
  输入: 异常信号 (模型超时/输出异常)
  输出: 降级/跳过/替换指令
  决策权: ★ (1/5 只在异常时介入)
```

**关键洞察**: 教练的"能力"不是单点质量，是这四个节点的**决策权平衡**。

如果 Pre-debate Coach 太强（策略完美）但 Round Whisper 太弱（不会动态调整）：
→ 辩论变成一个"照着稿念"的比赛，失去对抗性

如果 Pre-debate Coach 太弱（策略泛泛）但 Round Whisper 太强（频繁改写策略）：
→ 辩论方向每轮都在变，观众不知道正反方在主张什么

**架构要求**: L3.3 教练层需要明确的 **决策权重比**。建议 MVP 阶段 70/30（Pre-debate 七成，Whisper 三成），赛后分析根据实际效果调整。

## 2. 教练决策的反馈闭环

目前的教练架构缺少一个关键环节：**赛后反思**。

```
比赛前:
  Coach 输出 strategy_doc
  
比赛中:
  Coach 输出 round_whisper (x8)
  
比赛后:
  ⚠️ 没有赛后反思! Coach 不知道自己的策略在宏观上是否有效
```

赛后的 Coach 反馈是 L3.3 缺失的闭环：

```
赛后反思应该回答:
  1. "赛前策略中的哪些判断被事实证明是正确的？"
  2. "哪些判断是错的？"
  3. "下一场面对类似辩题时应该做什么调整？"
```

没有这个闭环，Coach 每一场都是从零思考，不会成长。而且**这是天然烧 token 的点**——每场比赛结束后的反思调用，每次都是几万 token 的独立推理。

**架构插入**: L3.3 教练层增加 `CoachReflection` 模块，在每场比赛评分完成后自动触发。输出 `coach_lessons.md`，附加到 tournament 记录中。

## 3. 教练策略的层级语言

教练输出的策略文档有一个被忽略的问题：**观众看得懂吗？**

```
典型的 strategy_doc:
  "攻击对方的逻辑基础——他们在 '自由意志' 的假设上站不住脚。
   从道家思想出发, 引用庄子的 '逍遥游' 来瓦解决定论的前提。
   在防守端, 守住 'AI 是对人脑的模拟而非替代' 这条底线。
   注意不要陷入 '意识是否存在' 的虚无争论。
   如果对方引用 '中文房间' 思想实验, 用 Searle 自己的修正来反驳。"

这条策略:
  ✅ Debater 能理解 (因为 Debater 是 AI)
  ❌ 观众不一定理解 (观众不知道 "中文房间" 是什么)
  ❌ 厂商数据分析时不好归类 (策略标签不结构化)
```

**架构建议**: Coach 输出应该分为三个层级：

```
Lv1: 观众摘要 (1句话)
  "本方策略: 先破对方逻辑根基，再守己方核心论点"

Lv2: Debater 指令 (完整策略文档)  
  [现有的 strategy_doc 内容]

Lv3: 战术标签 (结构化数据, 供赛后分析)
  {
    "attack_approach": "premise_undermining",
    "defend_approach": "boundary_maintenance", 
    "reference_intensity": "heavy",
    "risk_level": "moderate",
    "fallback_triggers": ["对方引用中文房间", "对方转向虚无主义"]
  }
```

三层输出的好处：
- Lv1: 观赛时右上角展示观众版策略摘要 → 增加观赛深度
- Lv2: 给 Debater 执行
- Lv3: 赛后分析——可以按战术标签聚合比赛: "所有使用 premise_undermining 策略的比赛, 正方平均得分 3.8"

## 4. 四对辩论的调度决策

4v4 辩论比 1v1 复杂一个数量级。调度决策维度：

```
对宫轮转顺序:
  Round 1: 乾宫 ↔ 坤宫 (正反一辩对决)
  Round 2: 巽宫 ↔ 震宫 (正反二辩对决)
  Round 3: 离宫 ↔ 坎宫 (正反三辩对决)
  Round 4: 兑宫 ↔ 艮宫 (正反四辩对决)
  Round 5-8: 自由辩论 (角色混合)

这里的架构选择:
  1. 固定轮转 (如上述) → 可预测, 观众容易跟上
  2. 自适应轮转 → 根据比赛状态动态调整:
     "乾宫被压得太惨了, 下一轮让巽宫 (更强的辩手) 上"
  3. 随机轮转 → 每场 shuffle, 增加不可预测性
```

**推荐**: V2.0 MVP 用固定轮转（简单、可预测），V2.1 引入自适应轮转（但自适应逻辑本身需要烧 token——每次调度的推理调用）。

## 5. 教练的知识产权问题

这是一个很少有人提但很重要的问题：**教练策略属于谁？**

教练策略是 Prompt + Topic + History 的产物。如果两家厂商都要求赛前教练策略公开（"我们想知道对手的策略是什么，以便针对性准备"），就产生了矛盾。

```
场景: 某厂商说:
  "既然是盲测, 教练策略也应该盲——我们不想让对手看到我们的比赛策略"

另一家厂商说:
  "教练策略是比赛的一部分, 应该公开, 就像围棋比赛后的复盘"
```

**架构回应**: 教练策略的可见性同样应该分级:

| 可见性等级 | 赛前看 | 赛中看 | 赛后看 |
|-----------|-------|-------|-------|
| 公开 | 可 | 可 | 可 |
| 参赛方可见 | 不可 | 不可 | 可 (解密后) |
| 隐藏 | 不可 | 不可 | 不可 (仅供内部数据分析) |

这个分级需要在 L4 产品层预设，在 L3.3 教练层的存储结构中预留 `visibility` 字段。

## 6. 教练决策树 vs 涌现策略

一个有趣的元问题：**什么情况下教练不应该做决策？**

这在现有架构中没有被考虑。教练默认"每轮都要指指点点"。

```
应该关闭教练的条件:
  ├─ 自由辩论阶段: 轮次本身没有预定结构
  │   └─ 教练的 Whisper 反而可能打断辩论的流动
  │
  ├─ 当辩手的遵从度 > 4.5/5 时:
  │   └─ 辩手已经严格执行策略, 不需要额外干预
  │
  ├─ 当辩手的遵从度 < 1.5/5 时:
  │   └─ 策略本身可能有问题, 不应该继续坚持
  │
  └─ 当比分差距 > 1.5 分时:
      └─ 胜负已定, 教练建议无法逆转
```

这些"不决策"的决策需要在 L3.3 中定义为 **Coach Inhibition Rules**。

---

## 结论

教练层不是"一个 AI 模型生成策略"，而是**一条有 4 个决策节点的流水线**，每个节点的决策权需要被明确定义和平衡。三个架构变化建议：

1. **赛后 Reflexion** — 增加教练反思闭环，每场赛后烧 token 做回顾
2. **三层策略输出** — 观众摘要 / Debater 指令 / 战术标签
3. **Coach Inhibition Rules** — 定义教练什么时候该闭嘴
