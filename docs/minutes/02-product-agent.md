# Product Agent · 价值传递层级分析

> **发言时间**: SESSION-001, 第二席
> **视角**: 从用户感知反推架构
> **核心问题**: 大厂打开这份邀约时，第一眼看到的是什么？

---

## 1. 大厂视角的价值逆推

先放下"我们能做什么"，从终点往回走。

### 终局画面

一位腾讯混元的 VP 收到一封邮件，标题是 "鲲鹏志 AI 辩论联赛 · 邀您参赛"。

**她 5 秒内做决定的部分：**

```
邮件标题: "鲲鹏志 AI 辩论联赛 · 邀您参赛"
  ├─ 5秒: "鲲鹏志是什么？" → 点不点开取决于:
  │    ├─ 是否知道你们 (品牌认知)
  │    │   └─ 不知道 → 邮件被归档, 邀约失败
  │    └─ 标题是否有趣 → "辩论联赛" 是有趣的
  │
  ├─ 30秒: 打开后看到什么？
  │    ├─ 一句话: "让您的模型在盲测中与全球顶尖模型同台竞技"
  │    │   └─ 吸引点: "盲测"、"同台竞技" → 想继续看
  │    ├─ 一张图: "上赛季的雷达图排名" → 如果图有说服力, 继续
  │    └─ 一段链接: demo 录像 → 如果 2 分钟内没感觉, 关闭
  │
  └─ 2分钟: 她想知道什么？
       ├─ "为什么我的模型应该参与？"
       ├─ "参与的代价是什么？"
       ├─ "我的竞品也参与了吗？"
       └─ "我会不会输得很丢人？"
```

**关键洞察**: 她从打开邮件到做决定，只有 2 分钟。这 2 分钟里，她用**产品直觉**评估的不是技术实力，而是**体面感**。

体面感 = "如果我推荐团队参加，我会不会看起来不专业？"

体面感的三个维度：
1. **视觉体面**: 雷达图好看吗？官网有设计感吗？
2. **数据体面**: 数据有说服力吗？分析深度够吗？
3. **叙事体面**: 这个故事我能在内部会议上讲吗？能写成 PR 稿吗？

## 2. 产品层级与用户感知的映射

```
架构层级          用户感知层级          体面要素
──────────────────────────────────────────────
L5 战略层  ───→  品牌叙事              "这是个正经事"
L4 产品层  ───→  用户体验              "看起来专业"
L3 系统层  ───→  数据可信度            "数据有说服力"
L2 模块层  ───→  功能完整度            "该有的都有"
L1 工具层  ───→  稳定性                "不会崩"
```

问题在哪里？**L4 到 L3 之间存在断裂。**

用户看到的是 L4 产品层（看板、雷达图、回放），但信服的是 L3 系统层（数据采集是否可靠、评分是否公正）。

如果 L4 展示一个完美的雷达图，但 L3 的评分逻辑没有公开文档，用户会直觉性地怀疑："这个评分是真实的吗？"

**架构应对**: L3 系统层的每个子系统（辩论引擎、教练、评分、存储）都需要对外输出一份「能力白皮书」——不是技术文档，而是面向厂商的"我怎么工作的"说明。

```
L3 辩论引擎 → 白皮书: "八股辩论的流程设计"
L3 教练层   → 白皮书: "教练策略的生成逻辑"
L3 评分层   → 白皮书: "六维雷达的评分标尺"
L3 存储层   → 白皮书: "数据的存证与验证机制"
```

这些白皮书是 L4 和 L3 的粘合剂——用户通过白皮书信服数据。

## 3. 产品层级的 3N 模型

任何层级的产品设计都可以用 3N 框架评估：

| 层级 | Notice (注意) | Navigate (导航) | Nudge (推动) |
|------|--------------|---------------|-------------|
| L4 观赛层 | 雷达图 | 点击维度展开详情 | "看，这个模型在战术执行度上排名前 3" |
| L4 参与层 | 厂商看板 | 筛选自己的模型 | "你的模型上周进步了 15%" |
| L4 治理层 | 赛事日程 | 排期、选辩题 | "下场比赛还有 3 天空位" |

3N 模型的核心洞察：**每一层都需要一个"推动"（Nudge），否则用户不会从"看看"变成"做点什么"。**

当前 L4 产品层缺少 Nudge 设计。厂商看板显示数据，但没有提示"接下来可以做什么"。

缺少的 Nudge 示例：
- "排名第 2，但你的反驳精准度只有 3.2，建议在 prompt 中加入更多逻辑链引导"
- "下场比赛 48 小时后开始，要不要提前提交辩题偏好？"
- "你的模型在 '历史借鉴' 维度下降了 0.3 分，可能最近赛后策略更新不够及时"

3N 不是 UI 设计问题，是产品层级的架构问题——**每层都要有"下一步"的出口，否则用户停留在当前层不会下沉。**

## 4. 赛事层级的反身性

赛事层级设计有一个反身性（Reflexivity）问题：

```
厂商参赛 → 比赛越激烈 → 数据越多 → 看板越有价值 → 更多厂商想参赛 → 赛事层级越丰富
```

但也有反向:

```
厂商不参赛 → 比赛少 → 数据稀疏 → 看板没说服力 → 更少厂商想参赛 → 赛事层级苍白
```

这是一个典型的**冷启动问题**。冷启动时的赛事层级设计决定了能否突破临界点。

**应对策略**: 赛事层级**从下往上建**：

```
Phase 0 (V2.0 内部测试):
  只有 1 个层级: 场次 (Match)
  参与者: 内部 8 个模型, 没有厂商
  目标: 验证比赛流程 + 数据产出

Phase 1 (V2.1 内部试运行):
  层级: 锦标赛 (Tournament) → 场次 (Match)
  参与者: 邀请赛 — 你认识的 2-3 个中腰部厂商
  目标: 验证厂商对接流程 + 看板 UX

Phase 2 (V3.0 公测):
  层级: 赛季 (Season) → 锦标赛 → 场次
  参与者: 开放注册, 首批 10 个厂商
  目标: 跑通完整的赛季制

Phase 3 (V4.0 联赛化):
  层级: 联赛 (League) → 赛季 → 锦标赛 → 场次
  参与者: 20+ 厂商, 常规赛 + 季后赛
  目标: 自运转
```

**关键设计**: 每层的设计都要"向上兼容"——Phase 0 的场次数据可以直接汇入 Phase 1 的锦标赛。这意味着 L3.6 存储层的数据结构必须从一开始就包含 `tournament_id` 和 `season_id` 字段（即使它们在 Phase 0 只是 null）。

## 5. 三种定价心智层级

面向大厂的邀约，本质上是一个**定价问题**。三种定价层级对应不同的产品价值主张：

| 心智层级 | 价值主张 | 邀约话术 | 适合厂商 |
|---------|---------|---------|---------|
| Lv1 参赛费 | "你来参赛" | "免费首场，入围联赛需缴纳年费" | 中腰部 |
| Lv2 赞助费 | "你是赞助商" | "你的品牌会出现在每场直播的片尾" | 头部 |
| Lv3 数据费 | "你是数据股东" | "所有比赛的数据都归你，你拥有专属分析报告" | 战略合作 |

这三个层级不是互斥的，而是递进的。同一个厂商可以从 Lv1 走到 Lv3。

**架构影响**: 不同的定价层级对应不同的产品功能暴露。

```
Lv1 参赛厂商看到:
  - 自己的 6 维雷达图
  - 和其他参赛模型的排名对比
  - 赛后回放

Lv2 赞助厂商额外看到:
  - 品牌的视觉露出
  - 赛季级的聚合数据（不暴露其他厂商细节）
  - 定制的赛后分析报告

Lv3 数据股东额外看到:
  - 全局数据（匿名化后）
  - 历史趋势分析
  - 模型能力的跨赛事追踪
```

这个数据可见性层级架构需要在 L4 产品层提前设计，而不是等厂商来了再加。因为 P0 阶段的 L1 数据字段影响 L3.6 存储层的 schema。

## 6. 用户流失预警层级

每个层级的用户流失，根因在不同层级：

| 用户行为 | 看到什么 | 为什么流失 | 问题出在哪层 |
|---------|---------|-----------|------------|
| 打开邀约邮件后关闭 | 邮件内容 | 品牌不熟悉, 缺乏信任 | L5 战略层 |
| 打开 demo 看了 30 秒关闭 | 辩论直播 | 不理解在干什么 | L4 产品层 (缺 5 秒引导) |
| 注册了但没参赛 | 厂商看板 | 不知道怎么接入 | L4 参与层 (缺 onboarding) |
| 参赛了一次没继续 | 赛后数据 | 数据太少, 不值得回来 | L3 评分层 (分析不够深) |
| 参加了一个赛季离开 | 赛季报告 | 对手不够多 | L4 赛事层 (网络效应不足) |

**层级响应**: 每个流失点都需要下层或者上层来回应。

---

## 结论

产品层级的本质不是 UI 设计，是**承诺与交付的匹配**。

L5 做出承诺（"这是最好的辩论评测平台"），L4 通过产品兑现承诺（"看，你的数据在这里"），L3 通过白皮书解释承诺（"评分是这么打的"），L2 和 L1 支撑承诺的实现（"没有 bug"）。

当前最大的产品层级断裂是：L4 展示的数据没有 L3 的白皮书解释，用户只能信任我们的黑箱。需在 V2.1 启动前补上 4 份系统白皮书。
