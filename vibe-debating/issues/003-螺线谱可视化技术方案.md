# Issue 003：螺线谱可视化技术方案

**GitHub Issue**: [Seek-Key-LTD/kunpengzhi-ai#3](https://github.com/Seek-Key-LTD/kunpengzhi-ai/issues/3)

## 背景

洛浦项目已有螺旋可视化原型（`12et.py`，matplotlib + pygame），但那是针对 MIDI 回放的被动展示。
Vibe Debating 需要主动式的 **螺线谱作曲/监看界面**。

## 技术约束

1. 最终产物是 Web 播放器的一部分（太虚幻境朋克风）
2. 需要实时响应 TTS 音频 + MIDI 演奏
3. 需要支持人机交互（拖拽作曲？标注？）
4. 跑在浏览器里，不能依赖 Python 后端渲染

## 候选方案

### 方案 A：Three.js / WebGL
- 3D 螺线管，可旋转
- 粒子系统表现音符流动
- 适合太虚幻境朋克的视觉风格
- 学习成本高

### 方案 B：Canvas 2D + WebAudio API
- 轻量，无依赖
- 基于洛浦 12et.py 的数学直接移植
- 与 WebAudio 的 AnalyserNode 天然配合
- 适合 MVP 阶段

### 方案 C：D3.js + SVG
- 交互性好，缩放拖拽
- 矢量图清晰
- 大量 DOM 节点可能卡顿

## 讨论要点

1. 第一版 MVP 应该上哪种方案？
2. 螺线谱是否支持用户在浏览器上拖拽作曲？还是只展示？
3. 螺线谱与 TTS 时间线的同步机制——如何让视觉随人声律动？
4. 太虚幻境朋克风的具体视觉调性（色彩、字体、粒子密度）？
